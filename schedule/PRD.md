## 1. 项目概述

### 1.1 项目背景
现有日程应用功能繁杂，学习成本高。用户只需要一个“快速记录、随时查看、可被AI调用”
的极简工具。本产品以**写起来快、看起来清、接起来易**为设计原则，同时提供开放的AP
I，方便与AI助手（如自定义GPT、Claude、个人脚本）集成。

### 1.2 产品目标
- **核心**：提供网页端日程管理界面，增删改查操作不超过3步。
- **开放**：提供RESTful API，允许AI工具或第三方脚本读写日程。
- **体验**：界面无干扰，单一聚焦日历与当日待办。

### 1.3 用户场景
- 个人快速记下明天10点会议。
- 通过AI助手口头添加“今天下午3点提醒我取快递”。
- 在手机浏览器打开网页，查看本周安排。

| 功能模块 | 功能点 | 描述 |
| :--- | :--- | :--- |
| 日程管理 | 添加日程 | 仅需填写标题、开始时间（日期+时间）。可选结束时间、备注
。 |
| | 编辑日程 | 点击日程修改任意字段。 |
| | 删除日程 | 支持单个删除或批量删除（一次勾选多个）。 |
| | 查看日程列表 | 以列表形式展示今日/明日/本周日程。 |
| 日历视图 | 月视图 | 网格显示月份，有日程的日期带圆点标记。点击日期切换到底部日
程列表。 |
| | 今日视图 | 点击“今天”返回当前日期，展示当天时间轴列表。 |
| 数据存储 | 后端持久化 | 所有日程存入数据库（SQLite/PostgreSQL）。 |
| API | RESTful API | 提供标准的CRUD接口，支持JSON格式，无复杂认证（初期使用静态
Token或免认证）。 |

### 后续版本（P1 – 增强体验）
| 模块 | 功能点 | 描述 |
| :--- | :--- | :--- |
| 提醒 | 浏览器通知 | 到达日程时间时弹出浏览器通知（需用户授权）。 |

### 暂不实现（P2）

- 文件/图片附件
- 自然语言解析（由AI调用API时负责解析，应用本身不内置）

## 3. 功能详细描述

### 3.1 日程数据模型（最小字段）
| 字段 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| id | 整数/UUID | 是 | 唯一标识 |
| title | 字符串(1~100) | 是 | 日程标题 |
| start_time | ISO 8601 字符串 | 是 | 例如 `2026-06-15T09:00:00+08:00` |
| end_time | ISO 8601 字符串 | 否 | 若为空，默认结束时间为开始时间后30分钟 |
| note | 字符串(0~500) | 否 | 备注 |
| created_at | ISO 8601 字符串 | 自动 | 创建时间 |
| updated_at | ISO 8601 字符串 | 自动 | 更新时间 |

**设计原则**：无重复规则、无标签、无提醒偏移 —— 将这些复杂逻辑交给外部AI或客户端
处理，本应用只做存储与展示。

### 3.2 网页界面交互
#### 布局（从上到下）
1. **顶栏**：显示当前年月，左右箭头切换月份，「今天」按钮。
2. **月视图区域**：7x6日历网格，每个格子显示日期数字，若当日有≥1个日程则显示一个
灰色圆点。
3. **日程列表区域**：位于月视图下方，展示所选日期的日程列表（按开始时间排序）。
每条日程显示标题、时间（开始~结束）。
4. **底部悬浮操作按钮**：「+」添加日程。

#### 添加/编辑弹窗（简洁表单）
- 标题（输入框，自动聚焦）
- 开始时间（本地datetime-local输入框）
- 结束时间（可选，datetime-local）
- 备注（多行文本，可选）
- 保存 / 取消按钮

#### 删除操作
- 日程列表中每条右侧有「...」菜单，点击出现「编辑」「删除」。
- 支持多选模式：点击列表左侧复选框 → 底部出现「删除所选」按钮。

### 3.3 API 设计（供 AI 工具调用）
**基础路径**：`/api/v1`  
**认证**（可选）：若需防滥用，使用静态Bearer Token（环境变量配置）。默认无认证，
仅允许本地/内网访问。

| 方法 | 路径 | 描述 | 请求示例 |
| :--- | :--- | :--- | :--- |
| GET | `/events` | 获取日程列表，支持时间范围查询 | `?start=2026-06-01&end=2026
-06-30` |
| GET | `/events/{id}` | 获取单个日程 | |
| POST | `/events` | 创建日程 | Body: `{"title":"开会","start_time":"2026-06-15T
09:00:00+08:00"}` |
| PUT | `/events/{id}` | 更新日程 | Body: 同创建，字段可选 |
| DELETE | `/events/{id}` | 删除单个日程 | |
| DELETE | `/events` | 批量删除 | Body: `{"ids":[1,2,3]}` |

**响应格式**：统一使用 JSON，成功返回数据，错误返回 `{"error": "message"}`。

**AI集成示例**：AI工具通过HTTP请求调用API，可将用户的自然语言“明天下午3点提醒我
写周报”解析为 `start_time=明天15:00`，然后POST到 `/events`。

## 4. 界面设计要求（简单整洁）

- **配色**：白色背景，浅灰分割线，主色为低饱和蓝色（`#3B82F6`）用于按钮和链接。
- **字体**：系统默认无衬线体（Inter / San Francisco / Roboto）。
- **圆角**：卡片、按钮圆角8px。
- **留白**：内容区域左右内边距16px，上下舒适。
- **无动画**：切换月份直接刷新，无过渡效果，保证快速响应。
- **响应式**：手机（≥320px）自适应，月视图自动缩小字体，日程列表单列显示。

## 5. 技术栈建议

| 层级 | 技术选择 | 理由 |
| :--- | :--- | :--- |
| 前端 | React + Vite + TailwindCSS | 快速开发，组件化，Tailwind保证简洁统一风格
 |
| 后端 | Node.js (Express) 或 Python (FastAPI) | 轻量，JSON处理方便，适合API服务
 |
| 数据库 | SQLite | 单用户场景，零配置，数据文件便携备份；未来可迁移至PostgreSQL
 |
| API文档 | Swagger / OpenAPI | 自动生成API文档，方便AI工具对接 |
| 部署 | Docker 或 直接运行在云服务器/VPS | 简单启动，也可使用Railway/Zeabur一键
部署 |

**前端额外库**：
- `react-calendar` 或手写简单日历网格
- `date-fns` 处理日期格式

## 6. 非功能性需求

| 类别 | 具体要求 |
| :--- | :--- |
| 性能 | 首页加载 <1.5秒（首次），API响应 <200ms（1000条日程内）。 |
| 并发 | 个人使用无需高并发，支持同一时刻最多5个请求。 |
| 数据安全 | 无用户系统，建议部署在内网或使用基本认证（HTTP Basic / Token）。API
支持CORS配置。 |
| 数据完整性 | 删除操作二次确认；数据库定期自动备份（可选脚本）。 |
| 可用性 | 日程时间不允许跨天（如开始 `23:00`，结束 `01:00` 视为非法，前端+后端
校验）。 |
| 可维护性 | 前端和后端分离，环境变量配置端口、Token。 |
| 跨平台 | 任何现代浏览器（Chrome, Safari, Edge 近两个版本）。 |

## 7. 开放API示例（供AI工具调用）

```bash
curl -X POST http://localhost:3000/api/v1/events \
  -H "Content-Type: application/json" \
  -d '{
    "title": "与设计师核对原型",
    "start_time": "2026-06-14T14:00:00+08:00",
    "end_time": "2026-06-14T14:30:00+08:00",
    "note": "会议链接稍后发"
  }'
```
